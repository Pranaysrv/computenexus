NexusTrace Library

Instrument your service from the inside. One line of middleware generates portable latency reports — consumable by the testing tool, your CI pipeline, or a plain text editor.

// install: npm i nexustrace
import nexusTrace from 'nexustrace'

// drop-in Express middleware — zero config
app.use(nexusTrace.middleware())

// generate report on demand
nexusTrace.generateReport({
  format: 'nexus',
  last: '5m',
  output: './reports/api-latency.nxr'
})
# install: pip install nexustrace
from nexustrace import instrument, NexusTrace

# FastAPI — decorator pattern
nt = NexusTrace(app=app)

@instrument
async def get_users():
  return await db.query("SELECT * FROM users")

# export .nxr report
nt.export(last="5m", path="api.nxr")
// go get github.com/computenexus/trace
import "github.com/computenexus/trace"

// wrap your ServeMux handler
nt := trace.New()
mux.Handle("/api/users",
  nt.Wrap(handler))

// generate report
nt.Report(trace.Options{
  Last: "5m",
  Out:  "api.nxr",
})

01
Inside-out measurement
Most tools measure from outside the service — they see network round-trip, nothing more. NexusTrace lives inside the process and captures every named span: middleware cost, DB wait, serialization, and flush time — separately and in microseconds.
02
Portable .nxr report file
Reports are self-contained binary files with a human-readable header. Email them, attach to a PR, diff against a previous version in CI — no dashboard login, no cloud dependency, no per-seat pricing.
03
Three generation modes
Always-on rolling buffer on disk (lock-free ring, near-zero overhead). On-demand snapshot via API call. Remote pull — the testing tool requests the buffer directly over the discovery channel without any file export step.
04
CI latency gates
Diff two .nxr files and fail the build if P99 exceeds the baseline. Latency regressions become first-class CI failures — the same way test failures are — before they reach production.

⇄
Visual ↔ DSL sync
Edit the visual canvas and the DSL updates instantly. Edit the DSL text and the canvas re-renders. Same paradigm Figma uses for properties — applied to API flows.
⇉
Parallel fan-out with barrier
Fire multiple API calls concurrently, define a barrier that waits for all branches, then merge their outputs into the next step. Measures the true critical path latency of parallel flows — not the sum.
⑂
Conditional routing
Branch flows based on response values using JSONPath conditions. If step 2 returns plan == "enterprise" route to 3A, otherwise 3B — no code required.
↺
Per-step retry policy
Define retries, backoff strategy (linear, exponential), and jitter per node. The latency report shows retry cost separately from the successful-path cost — so you see exactly what resilience is costing you.
⚡
Critical path analysis
After running a flow, ComputeNexus identifies which sequence of dependent calls drove the total time — and which calls could be parallelised to reduce it. Actionable, not just a waterfall chart.

Discovery layers
L1
Process Inspection
Reads the OS process table — /proc on Linux, libproc on macOS, ToolHelp32 on Windows — to find every process listening on a TCP port. Identifies runtime (Node.js, Python, Go, Java, Rust) from the binary path.
→ /proc · libproc · ToolHelp32
L2
Framework Introspection
Probes known introspection endpoints per framework: FastAPI's /openapi.json, Spring Boot's /actuator/mappings, Rails' route info endpoint. No code change required for these frameworks.
→ OpenAPI · Actuator · Rails · pprof
L3
Passive Traffic Sniffing
With optional packet capture permission, NexusProbe passively watches loopback traffic and builds a route inventory from observed requests — discovering endpoints as they are actually called, annotated with real latency distributions.
→ Optional · Local only · Loopback
L4
NexusTrace Beacon
When NexusTrace is embedded in the service, it broadcasts a structured mDNS announcement containing the service name, port, framework, and a compressed route manifest. The testing tool receives this within milliseconds of the service starting — no polling required.
→ mDNS · Sub-second · Route manifest
